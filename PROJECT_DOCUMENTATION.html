Section 01

Problem Statement

Understanding the gap we're addressing in text visualization

❌

The Problems

  • Long paragraphs are hard to process — Leading to low comprehension
  • Abstract concepts are difficult to visualize — Causing poor retention
  • No tool converts text to BOTH comics AND mind-maps — Gap in learning tools
  • Manual content transformation is time-consuming — Needs automation
✅

Our Solution

VisualVerse is a dual-mode system that:

  • 🔍 Automatically detects if input is a story or concept
  • 📚 Generates Comic Strips for narrative/story-based text
  • 🧠 Creates Mind-Maps for informational/conceptual text
  • ⚡ Processes in real-time with NLP pipeline

Complete NLP Pipeline Flow

Step 1: Text Preprocessing
Load SpaCy model → Tokenize text → Extract POS tags → Perform NER → Parse dependencies
def preprocess(text): nlp = spacy.load("en_core_web_sm") doc = nlp(text) tokens = [token.text for token in doc] pos_tags = [(token.text, token.pos_) for token in doc] entities = [(ent.text, ent.label_) for ent in doc.ents] return {"tokens": tokens, "pos_tags": pos_tags, "entities": entities}
Step 2: Text Classification
Extract 13 linguistic features (pronouns, filtered past tense, dialogue, explanation patterns, abbreviations) + TF-IDF → Random Forest Classifier → Classify as Narrative or Informational
# Classification Features (13 total): # Narrative: pronouns, past_tense (filtered), dialogue, said_verbs, story_words # Informational: definitions, bullets, numbers, technical_words, # explanation_patterns ("such as", "a subset of", etc.), # abbreviation_patterns ("(ML)", "(AI)", etc.) if narrative_score > informational_score: return "comic" # → Route to Comic Generator else: return "mindmap" # → Route to Mind-Map Generator
Step 3: Keyphrase Extraction (Hybrid NLP + ML)
Input from Preprocessing: entities, noun_chunks, nouns, subjects
ML (Gradient Boosting): 11 features (8 statistical + 3 NLP-derived) → Score & rank
# Uses PREPROCESSED DATA directly: entities = preprocessed["entities"] # From NER noun_chunks = preprocessed["noun_chunks"] # From SpaCy nouns = preprocessed["nouns"] # From POS subjects = preprocessed["subjects"] # From Dependencies # Feature Vector (11 features): features = [position, freq, length, spread, ...] # 8 statistical features += [is_entity, is_noun_chunk, is_subject] # 3 from preprocessing # Gradient Boosting Prediction: score = model.predict_proba(features) # P(is_keyphrase)
Step 4: Topic Modeling (LDA)
Input from Preprocessing: lemmas (stopwords removed)
LDA: Latent Dirichlet Allocation → Group keyphrases into topic clusters
# Uses PREPROCESSED LEMMAS: lemmas = preprocessed["lemmas"] # Base forms, stopwords removed # LDA Topic Modeling: lda = LatentDirichletAllocation(n_components=5) topic_dist = lda.fit_transform(lemmas) # Topic distributions
Step 5: Relation Extraction (Random Forest)
Input from Preprocessing: dependencies (nsubj, dobj, pobj)
Random Forest: Dependency path + TF-IDF → Classify relation type
# Uses PREPROCESSED DEPENDENCIES: dependencies = preprocessed["dependencies"] # Find keyphrase pairs with dependency paths: for dep in dependencies: child = dep["child"] # Subject head = dep["head"] # Object dep_type = dep["dep"] # nsubj, dobj, etc. # Random Forest Classification: relation = model.predict([features]) # IS_A, CAUSES, REQUIRES
Step 6: Output Generation
For Comics: Scene extraction → Character detection → Panel creation → Layout generation
For Mind-Maps: Build NetworkX graph → Calculate hierarchical layout → Create 3-level structure

Models & Training

What we trained, how we trained, and why

🧠

Text Classifier

Random Forest + TF-IDF

Training Data: Stories + Wikipedia articles

Purpose: Route to Comic or Mind-Map mode

Architecture: TF-IDF (500) + 11 Handcrafted Features → RF (100 trees)

Trained BOTH
🔑

Keyphrase Extractor

Gradient Boosting + NLP (Hybrid)

Training Data: Academic abstracts

Purpose: Extract keyphrases for Mind-Maps

NLP: NER + Noun Chunks + Dependencies + POS

ML: Gradient Boosting (100 trees) with 11 features

Trained MINDMAP Hybrid
📊

Topic Model

LDA (Latent Dirichlet Allocation)

Training Data: BBC News, WikiHow

Purpose: Group keyphrases into categories

Architecture: TF-IDF → LDA → Topic Distributions

Trained MINDMAP
🔗

Relation Extractor

Random Forest + Dependencies

Training Data: Annotated relation patterns

Purpose: Extract Subject-Verb-Object relations

Method: SpaCy Dependencies → RF Classification

Trained MINDMAP

Why Training Was Needed

Classifier: Pre-trained models don't differentiate story vs concept text — needed custom training

Keyphrase: Domain-specific extraction needed for accurate mind-map generation

Topics: Custom category grouping required for hierarchical mind-maps

Section 07

Backend API Flow

Step-by-step processing when you submit text

1. API Request Received
POST /api/process → Receives { text: "...", mode: "auto/comic/mindmap" }
2. Initialize NLP Components
TextPreprocessor, TextClassifier, KeyphraseExtractor, TopicModeler, RelationExtractor
3. Preprocess Text
preprocessor.process(text) → Returns tokens, POS tags, entities, dependencies
4. Classify (if mode="auto")
classifier.classify(preprocessed) → Returns "narrative" or "informational"
5A. Comic Generation (if narrative)
comic_generator.generate() → Scene extraction → Character detection → Panel creation
5B. Mind-Map Generation (if informational)
keyphrase_extractor.extract(20) → topic_modeler.model_topics() → relation_extractor.extract() → mindmap_generator.generate()
6. Return Response
{ mode, title, summary, comic_data OR mindmap_data }
